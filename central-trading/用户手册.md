## 1. 这个系统是干什么的


- 投资者发出买卖指令 → 系统收到
- 系统按照"价格优先、时间优先"的规则，自动把买家和卖家配对
- 配对成功就成交，成交结果通知买卖双方

**系统自动帮你处理的事情：**

| 事情 | 说明 |
|------|------|
| 连续撮合 | 交易时间内，买卖指令一来就立刻配对 |
| 集合竞价 | 开盘前把委托攒起来，到点统一撮合 |
| 涨跌停限制 | 普通股涨跌不超过±10%，ST股不超过±5%，超出的自动拦截 |
| 委托过期 | 收盘后没成交的委托自动作废，冻结的钱和股票自动解冻 |
| 行情推送 | 成交后第一时间把价格变动推送给客户端 |

## 3. 看板功能说明

登录后页面分为几个区域：

### 3.1 实时交易日志

这是看板的**核心区域**，所有交易事件会实时滚动显示。你能看到：

| 事件 | 颜色 | 什么意思 |
|------|------|---------|
| **委托** | 白色 | 有人下单了（买入或卖出） |
| **成交** | 绿色 | 撮合成功，买卖配对上了 |
| **撤单** | 橙色 | 有人取消了自己的委托 |
| **行情** | 蓝色 | 某只股票价格更新了 |
| **错误** | 红色 | 出了异常，需要关注 |
| **系统** | 灰色 | 系统状态变化（如进入集合竞价、收盘处理等） |

**操作技巧：**

- 想只看某一笔委托 → 在搜索框输入委托编号
- 想只看某个账号 → 在搜索框输入账号
- 信息滚太快 → 关掉"自动滚动"开关
- 屏幕太乱 → 点"清空日志"

### 3.2 委托簿查看器

输入**股票代码**（比如 `600519`），可以看到这只股票当前的买卖队列：

```
买盘（想买的人）              卖盘（想卖的人）
价格        数量              价格        数量
1850.00    1000股            1860.00     500股
1845.00    2000股            1870.00    1500股
1840.00    3000股            1880.00    2000股
```

- 买盘：价格从高到低，出价高的排前面
- 卖盘：价格从低到高，出价低的排前面
- 页面每5秒自动刷新

> 如果买一价 ≥ 卖一价，就说明有撮合空间，系统会自动成交。

### 3.3 股票注册

要新增一只可交易的股票：

1. 输入6位股票代码
2. 输入股票名称
3. 输入昨日收盘价
4. 点"注册股票"

> 注册后该股票即刻生效，可以接受委托。

### 3.4 连接状态

右上角有连接指示灯：

- **绿色"已连接"** → 正常，正在接收实时数据
- **红色"已断开"** → 连接断了，刷新页面即可重连

## 4. 撮合规则（通俗版）

了解这些规则有助于你看懂日志中发生了什么。

### 4.1 成交怎么定价？

系统取**买卖双方的中间价**来成交：

```
成交价 = (买方出价 + 卖方出价) ÷ 2
```

如果算出来的价格超出了涨跌停范围，就按边界价格成交。

**举例：**

- 张三出价 10.00 元买入
- 李四出价 9.80 元卖出
- 成交价 = (10.00 + 9.80) ÷ 2 = 9.90 元

### 4.2 谁先成交？

- **价格优先：** 买方出价高的先成交，卖方出价低的先成交
- **时间优先：** 价格一样的情况下，先下单的先成交

### 4.3 集合竞价怎么定价格？

开盘前的集合竞价阶段（默认 9:15-9:25），系统会找一个**能让成交量最大**的价格来统一成交。

如果有多个价格都能达到最大成交量，选**最接近昨日收盘价**的那个。

### 4.4 委托的7种状态

从下单到最终结果，一笔委托会经历以下状态：

```
已提交 → 已接受 → 部分成交 → 全部成交  （顺利的情况）
                 ↘ 已撤销              （投资者主动撤单）
                 ↘ 已过期              （收盘了还没成交）
                 ↘ 已拒绝              （价格超限、股票不存在等）
```

在日志中看到这些状态变化，就知道这笔委托经历了什么。

## 5. 日志怎么看

> **这是管理员最重要的日常工作。** 日志记录了系统所有行为，看懂日志就掌握了系统运行的全貌。

### 5.1 日志标签一览

每条日志开头都有一个方括号标签，告诉你这条日志来自哪个模块：

| 标签 | 负责什么 |
|------|---------|
| `[撮合成交]` | 买卖配对成功，产生成交 |
| `[撮合引擎]` | 撮合引擎的状态信息 |
| `[集合竞价]` | 集合竞价的撮合过程 |
| `[OrderService]` | 接收委托、验证委托 |
| `[TradeService]` | 成交后的结算处理 |
| `[ExpiryJob]` | 收盘后委托过期处理 |
| `[CallAuctionScheduler]` | 集合竞价自动触发 |
| `[PriceLimiter]` | 涨跌停价格检查 |
| `[AccountService]` | 资金账户操作（冻结/解冻） |
| `[Kafka]` | 消息发送和接收 |

### 5.2 正常业务日志（每天都会看到）

#### 收到委托

```
[OrderService] 收到委托: orderId=ORD20260624001, accountId=ACC001,
                stockCode=600519, side=BUY, price=1850.00, quantity=1000
```

**意思：** 账号 ACC001 下了一笔买入单，买 600519（贵州茅台），出价 1850 元，买 1000 股。这是最正常的业务日志。

#### 撮合成交

```
[撮合成交] BUY orderId=ORD001 vs SELL orderId=ORD002,
           stockCode=600519, tradePrice=1852.50, quantity=500
```

**意思：** 买方 ORD001 和卖方 ORD002 配对成功，成交价 1852.50 元，成交 500 股。绿色事件，一切正常。

#### 部分成交

```
[撮合引擎] 委托部分成交: orderId=ORD001, filledQuantity=300, remainingQuantity=200
```

**意思：** 这笔委托成交了 300 股，还剩 200 股在排队。说明当前对手盘不够，剩下的继续等。

#### 全部成交

```
[撮合引擎] 委托全部成交: orderId=ORD001
```

**意思：** 这笔委托的股票全部卖完（或买完）了，完结。

#### 撤单

```
[OrderService] 撤销委托: orderId=ORD001
[撮合引擎] 从委托簿移除: orderId=ORD001, stockCode=600519, side=BUY
```

**意思：** 投资者撤回了自己的委托，系统从撮合队列中移除了它。

#### 集合竞价

```
[CallAuctionScheduler] 进入集合竞价阶段
   ... （期间委托攒着不撮合） ...
[集合竞价] 撮合价格确定: stockCode=600519, auctionPrice=1855.00,
           最大成交量=12000, 成交笔数=28
[CallAuctionScheduler] 集合竞价结束，转入连续竞价阶段
```

**意思：** 集合竞价顺利完成。600519 以 1855 元成交了 12000 股，共 28 笔交易。

#### 收盘处理

```
[ExpiryJob] 开始执行日终过期处理, tradeDate=2026-06-24
[ExpiryJob] 过期委托数量: 15
[ExpiryJob] 释放冻结资金: accountId=ACC001, amount=1850000.00
[ExpiryJob] 日终过期处理完成
```

**意思：** 收盘了，有 15 笔委托没成交被标记为过期，对应冻结的资金和持仓已解冻。**每天收盘后必须检查是否出现这组日志。**

### 5.3 需要留意的日志

#### 委托被拒绝

```
[OrderService] 委托被拒绝: orderId=ORD20260624002, reason=价格超出涨跌停限制
```

**意思：** 有人下了偏离价格太远的单子。偶尔一两条正常，如果**频繁出现**，可能是客户端行情数据没及时更新。

#### 股票不存在

```
WARN [撮合引擎] 股票不存在: stockCode=999999
```

**意思：** 有人尝试交易系统中未注册的股票。需要确认是该注册这只股票，还是客户端发错了代码。

#### 价格超限

```
[PriceLimiter] 价格超出限制: stockCode=600519, price=2200.00,
               limitLow=1665.00, limitHigh=2035.00
```

**意思：** 有人出价 2200 元买茅台，但当天涨停价是 2035 元，被拒绝了。

### 5.4 看到这些日志要立即处理

#### 系统错误

```
ERROR [OrderService] 委托处理异常: orderId=ORD001, error=...
```

**意思：** 处理委托时系统内部出错。**立刻排查**，看是否影响其他交易。

#### 消息发送失败

```
[Kafka] 发送消息失败: topic=client.trade.report, error=...
```

**意思：** 成交报告发不出去，意味着**客户端可能收不到成交通知**。检查消息队列服务是否正常。

#### 账户系统断连

```
[AccountService] 调用账户系统失败: endpoint=/api/freeze, error=Connection refused
```

**意思：** 连不上资金账户系统。这会导致**无法冻结资金和结算**。如果系统配置的是真实模式（非Mock模式），需要紧急处理。

### 5.5 日志严重程度

| 级别 | 含义 | 你怎么做 |
|------|------|---------|
| INFO | 正常事件 | 日常扫一眼即可 |
| WARN | 需要注意 | 定期回顾，看频率是否异常 |
| ERROR | 出错了 | **立即处理** |

### 5.6 实战：通过日志排查问题

**"某笔委托为什么没成交？"**

在日志中搜索这笔委托编号，按顺序看：

1. 有没有 `收到委托` 日志？→ 没有说明委托根本没到系统
2. 有没有 `委托被拒绝`？→ 有就说明原因（价格超限、股票停牌等）
3. 有没有 `撮合成交`？→ 没有说明对手盘不匹配
4. 有没有 `撤销委托`？→ 投资者自己撤了
5. 有没有 `过期`？→ 收盘了还没成交

## 6. 日常操作

### 6.1 每天要检查的事

| 检查什么 | 怎么看 | 正常情况 |
|----------|--------|---------|
| 系统在不在运行 | 浏览器打开管理看板 | 能打开，连接灯绿色 |
| 收盘处理是否完成 | 日志搜 `日终过期处理完成` | 收盘后出现这条日志 |
| 集合竞价是否正常 | 日志搜 `集合竞价结束，转入连续竞价` | 开盘前出现 |
| 有没有严重报错 | 日志过滤 ERROR | 最好一条都没有 |

### 6.2 添加新股

在看板的"股票注册"区域，填写代码、名称、昨日收盘价，点击"注册股票"即可。

### 6.3 停牌 / 复牌

需要通过系统提供的管理接口操作。具体命令请参考运维团队提供的操作脚本。

停牌后：
- 这只股票**不再接受新委托**
- 已有的委托**仍可撤销**
- 复牌后恢复正常交易

### 6.4 手动管理集合竞价

正常情况集合竞价是自动触发的（默认 9:25）。

如果需要在非标准时间手动操作：
- **进入集合竞价阶段：** 新委托将进入集合竞价簿，暂时不撮合
- **触发撮合：** 对所有有委托的股票执行集合竞价撮合
- **查看状态：** 确认当前是集合竞价还是连续竞价阶段

> 具体操作命令请参考运维脚本。

### 6.5 手动触发收盘处理

正常情况收盘后系统会自动处理过期委托。如果需要手动提前触发，执行相应操作即可。

> **注意：这会立即把所有未成交委托作废并解冻资金，确认已收盘后再操作。**

## 7. 遇到问题怎么办

### 7.1 看板打不开

1. 确认服务器是否开机
2. 确认系统服务是否在运行（联系运维确认）
3. 检查自己的网络是否正常
4. 尝试刷新浏览器

### 7.2 客户端说收不到成交通知

1. 看板上能不能看到成交事件（绿色条目）？
   - 能看到 → 说明系统撮合正常，问题在于消息推送环节，联系运维检查消息队列
   - 看不到 → 说明可能委托本身没成交，按下面 7.3 排查

### 7.3 委托没成交

1. 在日志中搜索委托编号，确认委托是否被系统接收
2. 检查是否被拒绝（价格超限、股票停牌等）
3. 用委托簿查看器看这只股票的买卖盘，是否有对手盘
4. 检查是否处于集合竞价阶段（此阶段不进行连续撮合）
5. 检查该股票是否已停牌

### 7.4 成交价格看起来不对

记住系统用**中间价**成交，不是买方出价也不是卖方出价。

如果成交价明显偏离预期，检查：
- 成交价是否被涨跌停限制修正过（在日志中搜该股票的价格超限记录）
- 该股票昨日收盘价是否正确
