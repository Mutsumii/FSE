# FSE 股票交易系统用户手册

本手册面向本地演示、联调和日常使用人员，重点说明：

- 各子系统从哪里进入
- 用什么账号登录
- 实际能做什么
- 推荐的演示路径是什么

源码结构、环境变量、数据库初始化请看根目录 `README.md`。

## 1. 入口总表

| 子系统 | 使用对象 | 地址 | 说明 |
| --- | --- | --- | --- |
| 网上信息发布门户 | 所有人 | `http://localhost:3000/` | 系统导航页，也是行情门户入口 |
| 账户管理系统 | 工作人员 | `http://localhost:5173/login` | React 前端，调用 `8080` 后端 |
| 交易客户端 | 投资者 | `http://localhost:8090/` | 登录、下单、撤单、持仓与提醒 |
| 交易管理系统 | 管理员 | `http://localhost:8081/` | 黑名单、人工审核、权限维护 |
| 中央交易系统 | 管理员/演示人员 | `http://localhost:8082/` | 实时交易监控、订单簿、股票入库 |

## 2. 默认账号与登录说明

### 2.1 账户管理系统

- 仓库不自带默认工作人员账号
- 需要先在 `account_db.staff` 中插入工作人员
- 如果按根目录 `README.md` 的示例初始化，则可使用：
  - 账号：`staff001`
  - 密码：`123456`

首次登录会进入证书认证页面，认证码固定为：

```text
CERT-123456
```

### 2.2 交易管理系统

若已执行 `trade-management/sql/seed.sql`，默认可用：

- 账号：`admin`
- 密码：`admin123`

### 2.3 中央交易系统

监控页前端内置固定登录信息：

- 账号：`admin`
- 密码：`123456`

### 2.4 交易客户端

没有仓库内置的投资者演示账号，需要先在账户管理系统中创建：

- 证券账户
- 资金账户

然后使用资金账户登录交易客户端。若该资金账户第一次登录，也需要输入：

```text
CERT-123456
```

### 2.5 网上信息发布门户

- 直接打开即可，未登录时按游客模式展示
- 如果投资者先在交易客户端登录，再返回门户，门户会通过共享会话识别当前资金账户并切换为 `STANDARD`

## 3. 各子系统实际功能

### 3.1 账户管理系统

当前实际可完成：

- 工作人员登录与首次证书认证
- Dashboard 统计与操作日志查询
- 证券账户开户、挂失、补办、销户
- 投资者信息修改
- 资金账户开户、存款、取款、改密
- 资金账户挂失、补办、销户
- 资金账户与证券账户绑定/解绑
- 资金流水查询

当前业务约束：

- 必须先开证券账户，再开资金账户
- 证券账户有持仓时不能销户
- 资金账户挂失会联动冻结关联证券账户
- 资金账户销户后，失去绑定资金账户的证券账户会转入“无资金账户冻结”

### 3.2 交易客户端

当前实际可完成：

- 投资者资金账户登录
- 查询资金快照与证券持仓
- 查询股票行情
- 提交委托、撤单、查看成交结果
- 查看本地订单记录
- 价格提醒与通知

说明：

- 客户端页面由 `8090` 端口直接提供
- 其 Node 服务还会代理转发账户系统、中央交易系统、交易管理系统接口

### 3.3 交易管理系统

当前实际可完成：

- 管理员登录与注册
- 股票列表查看
- 按股票查看买卖委托
- 暂停/恢复股票交易
- 设置次日涨跌幅
- 黑名单新增、查询、移除
- 人工审核待处理委托
- 超级管理员查看管理员列表、维护角色和股票权限
- 超级管理员查看审计日志
- 在开启账户系统联调后，执行账户冻结/解冻

人工审核规则按当前代码为：

- 单笔委托金额大于 `100000.00` 元，进入人工审核
- 同一资金账户当日已自动通过 `5` 笔后，第 `6` 笔起进入人工审核
- 命中黑名单或股票不存在时直接拒绝

### 3.4 中央交易系统

当前实际可完成：

- 接收委托、撤单
- 撮合成交
- 提供订单反馈与成交结果查询
- 提供股票列表与单股详情
- 提供盘口快照
- 实时交易监控页
- 买卖队列查询
- 股票入库
- 集合竞价管理接口
- 日终超时委托处理接口

监控页中最常用的是：

- 实时日志滚动
- 搜索订单号/用户
- 查看某只股票的买卖队列
- 新增股票

### 3.5 网上信息发布系统

当前实际分为两部分：

- 前端门户：`http://localhost:3000/`
- 后端接口：`http://localhost:8083/api/publish`

前端可完成：

- 导航到其他子系统
- 查看股票搜索、行情、详情页
- 查看 K 线
- 发起 VIP 升级

按当前代码的权限差异：

- `GUEST`：可看基础行情，不可看 K 线，不显示盘口与主力信息
- `STANDARD`：可看常用 K 线周期 `5M/15M/30M/1H/1D`
- `PREMIUM_VIP`：解锁周/月/年 K 线

## 4. 推荐演示路径

### 4.1 工作人员开户与投资者交易

推荐顺序：

1. 打开 `http://localhost:5173/login`
2. 用工作人员账号登录账户管理
3. 首次登录时输入 `CERT-123456`
4. 新建证券账户
5. 为该证券账户开立资金账户
6. 给资金账户存款
7. 打开 `http://localhost:8090/`
8. 用刚创建的资金账户登录交易客户端
9. 查询股票并提交买入或卖出委托
10. 打开 `http://localhost:8082/` 查看中央交易监控页中的委托与成交变化

### 4.2 管理员审核与黑名单

推荐顺序：

1. 打开 `http://localhost:8081/`
2. 用 `admin / admin123` 登录
3. 查看股票列表和待审核记录
4. 添加黑名单用户，验证后续交易审核结果
5. 若已在 `config.properties` 中打开账户系统联调，再测试冻结/解冻账户

### 4.3 从门户统一进入各系统

推荐顺序：

1. 打开 `http://localhost:3000/`
2. 从导航页进入账户管理、交易客户端、交易管理、中央交易
3. 在交易客户端登录后返回门户
4. 再进入股票详情页，观察游客与登录用户展示差异

## 5. 常见使用问题

### 5.1 为什么账户管理页面能打开，但登不进去

最常见原因：

- `account_db.staff` 表里没有工作人员账号
- 密码不匹配
- 首次登录后没有完成证书认证

### 5.2 为什么交易客户端没有现成账号

因为交易客户端依赖账户管理中的真实资金账户。需要先在账户管理系统中完成开户，客户端才能登录。

### 5.3 为什么交易管理里的冻结/解冻按钮报错

因为 `trade-management` 默认 `account.enabled=false`。如果没有在 `config.properties` 中配置：

- `account.enabled=true`
- `account.api-base=http://localhost:8080`
- `account.staff-username`
- `account.staff-password`

那么管理员后台不会真正联到账户系统。

### 5.4 为什么网上信息发布后端启动后仍然查不到数据

常见原因：

- `stock_publish` 缺表
- Redis 未启动
- `central-trading` 未启动，导致同步不到股票与行情

### 5.5 为什么中央交易页能打开，但没有日志

页面能打开只说明 `8082` 服务已启动；没有日志通常表示：

- 还没有订单进入中央交易系统
- Kafka 链路被关闭，客户端没有通过 Kafka 推送
- 当前只在走 HTTP 接口，还没有触发成交广播

## 6. 启动后检查清单

启动全部服务后，建议按下面顺序检查：

1. 打开 `http://localhost:3000/`，确认导航门户可用
2. 打开 `http://localhost:5173/login`，确认账户管理前端可用
3. 打开 `http://localhost:8090/`，确认交易客户端可用
4. 打开 `http://localhost:8081/`，确认交易管理页可用
5. 打开 `http://localhost:8082/`，确认中央交易监控页可用
6. 在账户管理创建测试账户并充值
7. 在交易客户端登录并下单
8. 在中央交易与交易管理页面观察联动结果

## 7. 附：服务启动方法索引

详细启动命令见根目录 `README.md` 第 6 节。快速索引如下：

- 账户管理后端：`account-management -> mvn spring-boot:run`
- 账户管理前端：`account-management/frontend -> npm run dev`
- 中央交易后端：`central-trading -> mvn spring-boot:run`
- 交易管理后端：`trade-management -> mvn package` 后 `java -jar target/stock-trade-management-1.0.0.jar`
- 交易客户端：`trading-client -> npm start`
- 网上信息发布后端：`online-info-publish -> mvn spring-boot:run`
- 网上信息发布前端：`online-info-publish/publish-frontend -> npm run dev`
